## File Descriptions

### Root Level Files

#### `README.md`
The main entry point for the repository. Contains:
- Project overview
- Quick start instructions
- Installation guide
- Learning path overview
- Useful commands reference

#### `main.py`
The main entry point script. Currently imports from `week_2.lecture_2` module. Used to demonstrate how to organize and import modules from the project structure.

#### `pyproject.toml`
Poetry configuration file. Defines:
- Project metadata (name, version, description, author)
- Python version requirements (3.8.x)
- Direct dependencies (NumPy 1.22.2+)
- Development dependencies

Example:
```toml
[tool.poetry]
name = "python-template"
version = "0.1.0"
description = ""
authors = ["Your Name <you@example.com>"]

[tool.poetry.dependencies]
python = ">=3.8.0,<3.9"
numpy = "^1.22.2"
```

#### `poetry.lock`
Auto-generated lock file containing exact versions of all installed dependencies. Ensures reproducible builds across different machines.

#### `replit.nix`
Nix expression for setting up the development environment on Replit.

#### `.replit`
Configuration file for Replit online IDE integration.

#### `Git Commands.txt`
Quick reference for common Git operations used in this project.

### Week 2 Coursework

The `week_2/` directory contains organized coursework structured by lectures.

#### `week_2/lecture_1/`

**`part_c.py`** - Basic Variables and Print Statements
- **Purpose**: Demonstrate variable declaration and string output
- **Concepts**: Variables, data types, print function, string concatenation
- **Learning Goal**: Understand basic variable assignment and output operations

#### `week_2/lecture_2/`

**`part_b.py`** - Statistics from Student Marks
- **Purpose**: Calculate statistics from a list of student marks
- **Concepts**: Lists, loops, built-in functions (max, min, sum, len)
- **Learning Goal**: Work with collections and perform basic data analysis

**`part_c.py`** - Grade Calculation System
- **Purpose**: Convert numerical marks to letter grades
- **Concepts**: Lists, loops, conditionals (if/elif/else), string formatting
- **Learning Goal**: Implement decision logic and grade conversion rules

**`part_d.py`** - Sum Numbers Until Terminator
- **Purpose**: Sum a series of user-input numbers until a terminator value is entered
- **Concepts**: While loops, loop control (break), list operations, user input
- **Learning Goal**: Implement indefinite loops with termination conditions

## Module Organization

### Import Structure

The project uses Python's package structure. To import from this project:

```python
# In main.py or other scripts
from week_2.lecture_2 import part_d  # Import module
from week_2.lecture_1.part_c import fname  # Import specific variable
```

Each directory becomes a package if it contains an `__init__.py` file (or implicitly in Python 3.3+).

## Running the Exercises

### Method 1: Direct Script Execution
```bash
python week_2/lecture_1/part_c.py
python week_2/lecture_2/part_b.py
python week_2/lecture_2/part_c.py
python week_2/lecture_2/part_d.py
```

### Method 2: Using Poetry
```bash
poetry run python week_2/lecture_1/part_c.py
poetry run python week_2/lecture_2/part_b.py
```

### Method 3: Via Main Script
```bash
python main.py
```

## Environment Setup

### Virtual Environment Location
Poetry creates and manages the virtual environment in:
- **Linux/macOS**: `~/.cache/pypoetry/virtualenvs/[project-name]/`
- **Windows**: `%APPDATA%\pypoetry\virtualenvs\[project-name]\`

### Accessing Virtual Environment
```bash
poetry shell              # Activate in current terminal
poetry env info --path    # Show path to virtual environment
```

## Adding New Content

To add new exercises or lectures:

1. Create appropriate directory structure: `week_N/lecture_M/`
2. Add Python files: `part_a.py`, `part_b.py`, etc.
3. Update documentation files
4. Update this `STRUCTURE.md` file with new entries
5. Update `README.md` with learning path
6. Update `EXERCISES.md` with exercise descriptions
